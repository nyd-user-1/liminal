## Task 2 — Form 5500-SF load (NYS-146) ✅ DONE

Loaded the DOL/EFAST2 **Form 5500-SF** (short-form / small-employer) health
filings alongside the existing sql/040 registry. Extended, not rebuilt: a
**new sibling table** `form5500_sf_filings` (SF has no Schedule A and a
different column set — a separate table keeps the measurement clean and never
muddies the main-form × carrier join).

**Scope = HEALTH, mirroring the main form's 4A gate.** The main loader
qualifies on `TYPE_WELFARE_BNFT_CODE ~ '4A' OR a Schedule A health contract`;
SF carries no Schedule A, so the gate reduces to `SF_TYPE_WELFARE_BNFT_CODE ~
'4A'`. Pension-only SF filings (the overwhelming majority — small 401(k)s) are
skipped, same discipline as the main form skipping pension annuity contracts.

**Artifacts**
- `sql/040_form5500.sql` — appended `form5500_sf_filings` (idempotent
  `CREATE TABLE IF NOT EXISTS` + 3 indexes + comment; 50-line pure append,
  0 deletions). DDL applied to live Neon (19 columns confirmed).
- `scripts/ingest-form5500-sf.mjs` — new loader, single-pass, same psql-COPY /
  one-transaction / DISTINCT-ON-newest-DATE_RECEIVED idempotent pattern as
  `ingest-form5500.mjs`. `--year=YYYY [--dir=.harvest/form5500]`.
- Downloads + full-year loads ran **detached** (`.harvest/form5500/fetch-sf.sh`,
  `load-sf.sh` — both gitignored under `.harvest/`). 2025 was a foreground
  smoke test; 2023 + 2024 ran under nohup.

**EIN-join sanity (the acceptance) — how many NEW matchable EINs did SF add?**

→ SF and the main form are a **near-disjoint** population (0.6% overlap — by
design: SF = plans under ~100 participants, main = ≥100). The combined plan
registry grows **67,915 → 98,947 distinct EINs (+45.7%)**. This is exactly the
matchable-universe unlock the uhc-census probe predicted (the match ceiling was
set by the registry, not the matcher).

**Honest caveats**
- **State skew is structural, not a bug.** OH/GA/MO/KY = **86.6%** of SF health
  rows; **NY is only 185 rows / 98 EINs.** Fully-insured small welfare plans
  (<100 participants) are *exempt* from filing 5500/5500-SF, so what remains in
  SF-health is the self-funded / MEWA / association slice, which clusters in a
  few states. Spot-checked: state field maps correctly (NY rows are genuinely
  NY, plan names match sponsors, welfare codes 4A-prefixed). **Implication for a
  NY-focused census match: SF adds only ~98 NY EINs — the national lift is large,
  the NY lift is modest.** Worth stating before anyone assumes SF fills the NY
  gap.
- **Zero PHI.** Form 5500-SF is public DOL/EFAST2 data — employer names, EINs,
  plan names, participant counts, sponsor addresses. No patient data.
- No test rows to clean up — the SF data is intended production data; Task 1's
  refresh touched only the existing matview.
