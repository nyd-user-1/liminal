## What shipped

Liminal's version of Payr Advisors' three-page argument — *know your rates,
negotiate from evidence, dispute with the payer's own numbers* — with the one
decisive difference the brief demanded: **Payr shows one illustrative example;
we show the live corpus.** Every number is fetched at request time from the
database through `lib/repos/*`; nothing is hardcoded.

Three cross-linked public routes, under `app/(site)/` so they inherit the shared
`Nav` (ground `bg-page`) + `MarketingFooter`:

| Route | Job | The moment |
|---|---|---|
| `/pricing-data` | The corpus page | Live corpus stats band + the real 90837 payer spread |
| `/payer-negotiation` | The leverage page | The spread reframed as "why you're underpaid" + a truthful product description |
| `/payer-disputes` | The evidence page | An exhibit built from a real aggregate median + the NY DFS entity layer |

**Performance note (load-bearing):** an exact `count(*)` on the 13.6M-row fact
table took **15s**; distinct rated-NPI took **22s**. Both are unusable on a
request-time page. The fact-table figure uses `pg_class.reltuples` (**207ms**,
ANALYZE-backed) and renders as "13.6M+", so an estimate is both fast and honest.
Every other count is small and exact. All reads run in `Promise.all`; the page's
data fetch is bounded by the slowest (~1.3s for the distinct-tin org count).

## Honesty discipline (the four rulings)

- **Real data only** — 0 STAT placeholders in the rendered HTML; every figure
  traces to a repo read above.
- **No PHI / no client data** — public scope only (providers/orgs/payers/
  canonical entities/aggregate bands). Nothing touches clients, appointments,
  messages, notes.
- **No fabrication** — no testimonials, no "20–60%" claims, no outcome numbers.
  The exhibit is labeled "Illustrative", built from an aggregate median, and
  names only a public payer entity — never an org, provider, or claim.
- **Rate honesty** — every rate is framed "payer-published in-network median ·
  as of {date}"; the spread is explicitly aggregate market intelligence (NYS-35
  lane), never a plan × provider × code answer and never "what a patient pays".
