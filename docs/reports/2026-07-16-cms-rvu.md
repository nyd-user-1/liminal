## Decisions
- **Status = `A`+`T`, not `A` alone.** The release PDF says `A`,`R`,`T` "are used for Medicare payment". Dropped **R** — *"restricted coverage… if covered, contractor priced"*, so the formula does not produce what Medicare pays and a computed number there would be a confident lie (91% of R rows carry no RVUs anyway). This bites **exactly one seeded code, correctly**: `90846` (family therapy *without* the client present) is `R` — it has RVUs (2.74 work) but Medicare contractor-prices it, so it has **no benchmark row by design**, not a gap to backfill. `90847` (*with* the client) is `A` and present — the clinically meaningful comparison survives. `medicare_benchmark_ny` = 65 rows = 13 codes × 5 localities.
- **Non-APM CF** — which CF applies is a property of the *billing party*, not the code, and nothing in `provider_rate_signals` says who cleared QPP thresholds.
- **`cms_gpci` PK = (state, locality_code, year).** The brief's `locality_code` alone is not unique — `00` is Alabama *and* Arizona, `01` spans 19 states; keying on it collapses 109 localities to ~30.
- **RVU26C (rel. 06/30/2026), not the RVU26B the brief expected** — releases *restate*, not append.
- **579 modifier records excluded** from `hcpcs_codes`: blank code positions, 2-char modifier at 4–5 (`A1` = "Dressing for one wound"). Different entity; loading blind inserts rows keyed `"   A1"`.
- **`cpt_codes` wording is DRAFT, editable content** — all 14 strings ours; the five live ones match `RATE_CODES` (`lib/rate-table.ts:25`) exactly.

## Gotchas
- **CMS 403 did not reproduce** — default curl got 200 on both pages today. Browser UA sent anyway.
- **A code is not a record (HCPCS).** Descriptors >80 chars continue onto further records (rec id 3 = first, 4 = continuation); 4,108 of 8,725 reassemble. Reading only the first yields a sentence chopped mid-clause — which looks like data, not like a bug.
- **PPRRVU quotes comma-bearing fields** (`"Asy carbamazepin 10,11-epxid"`, 1,037 lines) — naive `split(",")` misaligns them. Both header rows are located **by content**, never index: the junk-row count has changed across releases, and a shifted offset loads RVUs into wrong columns silently.
- **GPCI has 6 footnote rows** (value in col 1, no state) — filter on col 1 and you get 115 "localities". One confirms the 1.0 work-GPCI floor runs through 2027-01-01, so CY2026 already includes it.
- **Benchmark is Manhattan; the rates are statewide.** Rest-of-NY runs ~10% below Manhattan, so a Rest-of-NY provider's true % of Medicare is ~10 points *higher* than the table shows. Locality-aware comparison needs a county → Medicare-locality mapping — not built.
- `short_description` is often useless for H-codes (H0004's is "Alcohol and/or drug services"; the *long* one says behavioral health counseling). Prefer `long_description`. NLM Clinical Tables is Level II only and returns only the short text — re-probed: `90791` → 0 hits, `H0004` → 1.
