### 1 ‚Äî NYS-119 ¬∑ "died" cap vs harvest timeouts (low-medium)
`lib/repos/sync-runs.ts` `DIED_MINUTES = 30` is right for the Vercel matview cron
(killed at 300s) but is applied uniformly to `harvest:*` rows, whose jobs carry
`timeoutMinutes: 600`. Confirmed the render path: `sync-health.tsx` maps
`DOT.died ‚Üí "danger"` in the "Harvest runs" list. Display-only (no false
email/bell ‚Äî those fire on a job's real non-zero exit), so it's a misleading
dashboard state during the long harvests the card exists to watch, not a false
alert. Fix in the issue: threshold per-job-type from `timeoutMinutes`, or exempt
`harvest:*` and trust their self-reported close.

### 2 ‚Äî NYS-117 ¬∑ manifest-name shell interpolation (low)
`runner.mjs` `manifestJobs()` interpolates `name = file.replace(/\.txt$/,"")`
into a command run via `spawn("/bin/bash", ["-c", job.run])` ‚Äî unquoted in
`.harvest/mrf/${name}/*.csv` and inside quotes in the runner args. Threat model
is low (queue-dir write already = local code execution), so this is mostly a
**robustness** defect: a manifest named `mvp behavioral.txt` silently breaks the
glob and the args. Fix: validate the stem `^[A-Za-z0-9._-]+$` ‚Äî the discipline
`nppes-weekly.sh` already uses on the filenames it greps off CMS.

### 3 ‚Äî NYS-118 ¬∑ CPT label drift (low)
`cpt.ts`'s own header says "keep the two in sync"; they aren't. Verified against
the live DB ‚Äî 4 of the 6 sql/050 codes have different wording in `cptLabel` than
in `cpt_codes.display_name`. Same billing code renders two names depending on the
/rates path; tonight's rescans make it visible. Fix: align, or generate `cpt.ts`
from `cpt_codes` (same single-source spirit as NYS-115).

## Verified CLEAN (with evidence)

**Auth on the new API surface.**
- `app/api/notifications/route.ts` ‚Äî GET/POST both `requireUser()`; every repo
  query scoped `WHERE user_id = ${userId}` (parameterized). No IDOR: a user reads
  and marks only their own rows. No PHI (schema comment + producers write job
  names only).
- `app/api/cron/daily/route.ts` ‚Äî `CRON_SECRET` guard: **unset ‚áí 503** (closed
  door), wrong/absent ‚áí 401. The new `notifyAdmins` on failure is `.catch(()=>{})`
  best-effort.

**Injection surface of the psql subprocesses + `spawn` patterns.**
- `scripts/ingest-form5500.mjs` ‚Äî data reaches Postgres over **COPY (the data
  channel), never SQL string interpolation**; column lists are static constants;
  `--year` validated `^\d{4}$`. Every CSV field is quote-wrapped + `"`‚Üí`""`
  escaped, and rows carry 26/28 columns, so a bare `\.` COPY terminator can't be
  forged. `spawn("psql", [DB, ‚Ä¶])` ‚Äî no shell.
- `.harvest/status.mjs` ‚Äî all queries static except a `payer_sources.id` **uuid**
  read from the DB and inlined (no quote chars possible); `spawn("psql", ‚Ä¶)`, no
  shell; `ON_ERROR_STOP=1`. The `count(p.*)‚Üícount(p.id)` rewrite is semantically
  identical (p.id is the non-null PK on matched LEFT-JOIN rows) ‚Äî verified.
- `ops/harvest/tasks/nppes-weekly.sh` ‚Äî the downloaded filenames are constrained
  by a strict regex (`[0-9]{6}_[0-9]{6}_Weekly_V2\.zip`) before use; `"${args[@]}"`
  quoted; `set -euo pipefail`. `ops/harvest/install.sh` ‚Äî only static
  `com.liminal.harvest` / `gui/$(id -u)` constants to `launchctl`.

**Secret handling.** `LIMINAL_RESEND_API_KEY` is used in the Resend Authorization
header and **never logged** (runner logs `e.message` only); `lib/email.ts` no-ops
(returns false) without the key. Failure emails/notifications carry job + table
names only.

**Failure modes.**
- DB down ‚Üí `runner.mjs` `sql` is null ‚áí ledger + bell inserts are graceful
  no-ops; the job itself fails and is emailed/retried. The cron's `openRun`/
  `closeRun` swallow errors ("the log is a convenience, not the job").
- Resend down ‚Üí `emailFailures`/`sendEmail` wrap `fetch` in try/catch, log, and
  return; the calling flow is never broken.
- Payer host down ‚Üí the harvest job's `curl -f` exits non-zero ‚áí job marked
  failed ‚áí retried (2√ó, 120s backoff) ‚áí emailed; KILL SWITCH short-circuits
  retries on a DB write error.
