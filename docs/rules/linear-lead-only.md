# Linear is lead-only

Issues are filed, closed and reorganised by the lead session alone. Every other terminal reports its Linear intents; one hand keeps the board coherent.

**Why it exists.** Ten agents filing freely produces ten parallel taxonomies, duplicate issues for the same defect, and a board that describes the agents' inner lives rather than the product's state. Structure is a single-writer problem.

**How to apply.** Do the work, then report what you believe should be filed, closed, or moved — with the reasoning. The lead rules on it. Never close another session's issue, and never close a safeguard as moot because it was deferred: a removed safeguard is a regression and stays open.
