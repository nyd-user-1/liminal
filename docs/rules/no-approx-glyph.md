# No almost-equal glyph

Estimates read as "14M+" or "155,317+", never the almost-equal sign. An exact count and an estimate should look like the same kind of thing, one just carries a "+".

**Why it exists.** The almost-equal glyph reads as an apology and renders inconsistently across the fonts in the stack. A trailing "+" carries the same information — this is a floor, not a ceiling — while keeping the figure scannable next to its exact neighbours.

**How to apply.** Floor the estimate, format it like any other number, append "+". Never mix the two conventions in one row of cards.
